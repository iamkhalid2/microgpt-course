    import os       # os.path.exists

    Three standard Python libraries, zero external dependencies. The random.seed(42) ensures that every time you run the file, you get the same "random" numbers ‚Äî making experiments reproducible.

    Problem ‚Üí Solution

    Problem: How do we figure out which parameters are responsible for errors?

    Solution: Wrap every number in a Value object that remembers how it was computed. Then walk backwards through the computation to assign blame.

    This is the heart of the file. The Value class is a tiny automatic differentiation engine ‚Äî the same idea behind PyTorch's autograd.

    Block 5: The Architecture (Lines 92‚Äì144)¶

    microgpt.py ‚Äî Lines 92-144 (simplified)
    def gpt(token_id, pos_id, keys, values):
    # 1. Look up embeddings
    # 2. For each layer:
    #    a. Multi-head attention (look at context)
    #    b. MLP (process information)
    # 3. Output logits (raw scores for each possible next character)
    return logits

    Problem ‚Üí Solution

    Problem: Given the current character and position, how do we compute a prediction?

    Solution: A pipeline of transformations: embed ‚Üí normalize ‚Üí attend ‚Üí think ‚Üí predict.

    This is the Transformer architecture ‚Äî the "T" in "GPT".

    Covered in detail in Module 3
